# 总结

可变数据类型会让程序更难推理。例如，在学习 ref 之前，
我们不需要担心 OCaml 中的别名问题。但可变性确实有其用途。
I/O 本质上就涉及状态变化。还有一些数据结构，例如数组和哈希表，
如果没有可变性就无法高效实现。

因此，可变性提供了强大的力量，但力量越大，责任越大。
尽量不要滥用你刚获得的力量。
