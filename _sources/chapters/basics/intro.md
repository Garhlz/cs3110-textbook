# OCaml 基础知识

本章将介绍 OCaml 的一些基本特性。但在深入学习 OCaml 之前，让我们先讨论一个更大的话题：如何学习编程语言。

本课程的一个重要目标，不仅是让你学会一门新的编程语言，更是提升你**学会学习**新语言的能力。

学习语言有五个基本要素：语法、语义、习语、库和工具。

**语法。** *语法*定义了什么样的文本在该语言中构成良好形成的程序，包括关键字、空白与格式的规则、标点符号、运算符等。学习新语言时，如果语法与你已知的语言差异很大，可能会感到烦恼。但学的语言越多，你就越能接受每种语言自有的语法，而不是希望它们都相同。（想看看真正不寻常的语法吗？可以看看需要特殊键盘的 [APL][tryapl] 和程序完全由空格、制表符和换行符组成的 [Whitespace][whitespace]。）要与计算机交流，你必须理解语法。

**语义。** *语义*定义了程序的行为规则。换句话说，语义是关于程序的**含义**——特定的语法代表什么样的计算。请注意，虽然"语义"（semantics）在形式上是复数，但我们将其作为单数使用，类似于"数学"（mathematics）或"物理学"（physics）。

语义有两个方面：语言的*动态*语义和*静态*语义。动态语义定义了程序在运行时如何执行和求值。静态语义定义了编译时进行的检查，以确保程序合法，这超出了语法检查的范围。最重要的静态语义是*类型检查*：确定程序是否类型正确的规则。学习一门新语言的语义通常是真正的挑战，尽管语法可能是你首先遇到的障碍。你需要理解语义才能向计算机表达你的意思，并且只有表达清楚，程序才能正确执行计算。

**习语。** *习语*是指使用语言特性来表达计算的惯用方式。在一种语言中，你可能有多种方式表达同一个计算，你会选择哪一种？有些方式会比其他方式更自然。精通该语言的程序员会更偏爱某些表达方式。我们可以从有效使用语言中主导范式的角度来理解这一点，无论这些范式是命令式、函数式、面向对象的，还是其他风格。你需要理解习语，才能不仅向计算机、也向其他程序员准确表达你的意思。用地道的方式编写代码，其他程序员会更容易理解你的代码。

**库。** *库*是已经为你编写好的代码集合，可以让你成为更高效的程序员，因为你不必自己编写这些代码。（有句话说[懒惰是程序员的一种美德][lazy]。）学习新语言的一部分是了解有哪些可用的库以及如何使用它们。每种语言通常都提供一个*标准库*，让你能访问一组核心功能，其中许多功能你自己用该语言是写不出来的，比如文件 I/O。

**工具。** 任何语言实现至少都提供编译器或解释器，作为使用该语言与计算机交互的工具。但还有其他类型的工具：调试器、集成开发环境（IDE）、以及用于性能、内存使用和正确性等方面的分析工具。学习使用与语言相关的工具同样能让你成为更高效的程序员。有时容易把工具本身与语言混淆；例如，如果你只在 Java 项目中使用过 Eclipse，你可能不知道 Eclipse 是一个支持多种语言的 IDE，而 Java 完全可以在没有 Eclipse 的情况下使用。

在本书中学习 OCaml 时，我们的重点主要是语义和习语。当然，我们会一路学习语法，但那不是最有趣的部分。我们会接触 OCaml 标准库和其他几个库，特别是 OUnit（一个单元测试框架，类似于 JUnit、HUnit 等）。除了 OCaml 编译器和构建系统，我们主要使用的工具是顶层（toplevel），它提供了交互式实验代码的能力。
