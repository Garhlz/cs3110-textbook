# 并发

并发程序让多个计算在时间上发生重叠，而不是被迫完全按顺序执行。

* 例如，*图形用户界面* (GUI) 依靠并发性来保持
用户界面有响应，同时计算在后台继续进行。如果没有并发，GUI 会“锁死”，
  直到当前操作完成。有时由于并发错误，仍然会发生这种情况&mdash;
  这会让用户非常沮丧！

* 电子表格需要并发性来重新计算所有单元格，同时保持菜单和编辑功能可用。

* Web 浏览器需要并发性来随着新数据通过网络传入而增量读取和渲染网页，
  运行嵌入在网页中的脚本，并让用户能够浏览页面、点击超链接。

*服务器*是另一类需要并发的应用程序。Web 服务器需要响应许多客户端请求，
而客户端通常不愿意等待。例如，如果作业发布在 CMS 上，你肯定希望能和
班上其他人同时查看作业，而不是像在车管所或老式熟食店那样“取号”等叫号。

*操作系统* (OS) 的主要工作之一是提供并发性。
该操作系统使许多应用程序可以同时执行：
音乐播放器、网络浏览器、代码编辑器等。它是如何做到这一点的？有
两种基本的、互补的方法：

* **交错：**在计算之间快速来回切换。对于
例如，执行音乐播放器 100 毫秒，然后执行浏览器，然后
  编辑器，然后重复。这使得它看起来像是多次计算
  是同时发生的，但实际上，只有一个发生在
  同时。

* **并行性：**使用能够执行两个或多个任务的硬件
实际上是同时进行计算。如今许多处理器都是
  *多核*，意味着它们有多个中央处理单元 (CPU)，
  每个都可以同时执行一个程序。

无论使用哪种方法，并发编程都是具有挑战性的。
即使有多个核心可供同时使用，仍然有
许多其他必须共享的资源：内存、屏幕、网络
管理该共享，尤其是在不引入错误的情况下，是
相当困难。例如，如果两个程序想要使用
计算机的内存中，需要就每个程序的运行时间达成某种协议
允许从内存中读取和写入。否则，例如，两个程序
可能会尝试写入内存中的同一位置，从而导致损坏
数据。那些类型的“竞争条件”，程序竞相完成其任务
众所周知，在另一个程序之前进行操作是很难避免的。

最基本的挑战是，并发性会让程序执行变得*非确定性*：
操作发生的顺序不一定能提前知道。竞争条件就是非确定性的一个例子。
为了在非确定性面前正确编程，程序员不得不考虑“所有”可能的操作顺序，
并确保程序在*所有*这些顺序下都能正常工作。

纯函数式程序使非确定性更容易推理，因为
无论如何，表达式的求值总是返回相同的值。对于
例如，在表达式 `(2 * 4) + (3 * 5)` 中，可以执行操作
同时（例如，同时求值左右两个乘积）
而不改变答案。命令式编程的问题更多。对于
例如，表达式 `!x` 和 `incr x; !x` 如果同时执行，可以
根据先执行的结果给出不同的结果。

## 线程

为了使并发编程变得更容易，计算机科学家发明了许多
抽象。最著名的之一是“线程”。抽象地讲，线程是一个
单个顺序计算。一次可以有多个线程运行，
取决于硬件，多个线程可以交错运行或并行运行；*调度器*
负责选择在任意给定时间运行哪个线程。调度可以是“抢占式”的：
调度器可以停止线程并稍后重新启动它，线程自己对此没有选择权；
也可以是*协作式*的：线程必须主动选择把控制权交还给调度器。
前者可能导致竞争条件，后者可能导致应用程序无响应。

具体来说，线程是加载到线程中的一组值
处理器的寄存器。  这些值告诉处理器在哪里可以找到
下一条要执行的指令，其堆栈和堆位于
内存等。为了实现抢占，调度器在内存中设置一个定时器
硬件；当定时器关闭时，当前线程被中断，
调度程序开始运行。系统编程和操作系统课程
通常会详细介绍这些概念。
