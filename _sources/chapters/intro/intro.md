# 通过 OCaml 更好地编程

你已经会用 Python 或 Java 这样的主流语言编程了吗？很好。这本书就是为你准备的。现在是时候学习如何编写更好的程序了。是时候学习一门函数式语言：OCaml。

函数式编程提供了一种不同于你之前所知道的编程视角。适应这种视角需要放下一些旧观念：赋值语句、循环、类和对象，等等。这并不容易。

> <i>南隐（Nan-in）是明治时代（1868-1912）的一位日本禅师。有一天，一位大学教授来向他请教禅。
> 南隐为客人斟茶。他把客人的杯子倒满，却仍继续倒下去。
> 教授看着茶水溢出，终于忍不住说：“已经满了，再也装不下了！”
> 南隐说：“就像这个杯子一样，你心里装满了自己的意见和猜测。
> 如果你不先倒空自己的杯子，我又怎么能向你展示禅呢？”</i>

我相信学习 OCaml 会让你成为一名更好的程序员。原因如下：

- 你将体验*不可变性*的自由：所谓"变量"的值不能改变。再见，调试时的噩梦。

- 你会提升*抽象*能力，即通过提取共性来避免代码重复。再见，臃肿的代码。

- 你将接触到一个*类型系统*。一开始你可能会讨厌它，因为它拒绝那些你以为正确的程序。但你最终会喜欢它，因为你会意识到它是对的，而你的程序错了。再见，失败的测试。

- 你将接触到*编程语言的理论与实现*，从而理解自己写代码时究竟在告诉计算机什么。再见，神秘而奇异的咒语。

所有这些思想也可以在其他语境和语言中学习。但 OCaml 提供了一个绝佳的机会，将它们全部汇集在一起。**OCaml 将改变你对编程的思考方式。**

```{epigraph}
“一门不会影响你思考编程方式的语言，不值得学习。”

-- Alan J. Perlis（1922-1990），第一位图灵奖获得者
```

此外，OCaml 很美。它优雅、简单、从容。审美很重要。代码不只是写给机器执行的，也是写给人阅读的。优雅的代码更容易理解和维护，尽管写出来并不一定更容易。

你写出的 OCaml 代码可以有风格、有品味。起初这可能并不明显。毕竟你正在学习一门新语言，就像不会有人期待在"梵文入门"的第一天就能欣赏梵文诗歌一样。事实上，在努力用一门新语言表达自己的过程中，你很可能会沮丧一阵子。给自己一些时间。掌握 OCaml 之后，当你再回到已经熟悉的其他语言时，也许会惊讶地发现它们看起来多么笨拙。
