练习

4.9. 练习#

Solutions to most exercises are available. We are happy to add or correct solutions. Please make contributions through GitHub.

Exercise: twice, no arguments [★]

考虑以下定义：

let double x = 2 * x
let square x = x * x
let twice f x = f (f x)
let quad = twice double
let fourth = twice square

使用顶层来确定 quad 和 fourth 的类型。解释一下 quad 在语法上怎么可能没有写成一个函数 一个参数，但它的类型表明它实际上是一个函数。

Exercise: mystery operator 1 [★★]

下面的运算符是做什么的？

let ( $ ) f x = f x

提示：调查 square $ 2 + 2 与 square 2 + 2。

Exercise: mystery operator 2 [★★]

下面的运算符是做什么的？

let ( @@ ) f g x = x |> g |> f

提示：调查 String.length @@ string_of_int 应用于 1、10、100、 等

Exercise: repeat [★★]

将 twice 概括为函数 repeat，使得 repeat f n x 适用于 f 到 x 总共 n 次。也就是说，

  • repeat f 0 x 产生 x

  • repeat f 1 x 产生 f x

  • repeat f 2 x 产生 f (f x) （与 twice f x 相同）

  • repeat f 3 x 产生 f (f (f x))

  • …

Exercise: product [★]

使用 fold_left 编写一个函数 product_left 来计算以下乘积 浮动列表。空列表的乘积是1.0。 提示：回想一下我们是如何 在讲座中仅用一行代码就实现了 sum。

使用 fold_right 编写计算乘积的函数 product_right 浮点数列表。 同样的提示适用。

Exercise: terse product [★★]

你可以为 产品 练习制定多简洁的解决方案？ 提示：你 每个只需要一行代码，并且不需要 fun 关键字。对于 fold_left，你的函数定义甚至不需要显式地采用 列表参数。如果你使用 ListLabels，则 fold_right 也是如此。

Exercise: sum_cube_odd [★★]

编写一个函数 sum_cube_odd n 来计算所有的立方之和 0 和 n 之间的奇数（包括 0 和 n）。不要编写任何新的递归 函数。相反，请使用map、fold 和 filter以及 ( -- ) 运算符（在流水线讨论中定义）。

Exercise: sum_cube_odd pipeline [★★]

重写函数 sum_cube_odd 以使用管道运算符 |>。

Exercise: exists [★★]

考虑编写一个函数 exists: ('a -> bool) -> 'a list -> bool，这样 exists p [a1; ...; an] 返回列表中是否至少有一个元素 满足谓词 p。也就是说，它的求值结果与 (p a1) || (p a2) || ... || (p an)。当应用于空列表时，它会计算 至 false。

正如我们上面所做的那样，写出这个问题的三个解决方案：

  • exists_rec，必须是不使用List的递归函数 模块,

  • exists_fold，它使用 List.fold_left 或 List.fold_right，但是 不是任何其他 List 模块函数，也不是 rec 关键字，并且

  • exists_lib，它使用 List 模块函数的任意组合，除了 fold_left 或 fold_right，并且不使用 rec 关键字。

Exercise: account balance [★★★]

编写一个函数，给定代表借方的数字列表，扣除 从账户余额中扣除，最后返回账户中的剩余金额 平衡。编写三个版本：fold_left、fold_right 和直接递归 实现。

Exercise: library uncurried [★★]

这是 List.nth 的未柯里化版本：

let uncurried_nth (lst, n) = List.nth lst n

以类似的方式编写这些库函数的非柯里化版本：

  • List.append

  • Char.compare

  • Stdlib.max

Exercise: map composition [★★★]

展示如何替换 List.map f (List.map g lst) 形式的任何表达式 使用仅调用 List.map 一次的等效表达式。

Exercise: more list fun [★★★]

编写执行以下计算的函数。你所使用的每个函数 write 应使用 List.fold、List.map 或 List.filter 之一。来选择 使用哪一个，想想计算正在做什么：组合， 转换或过滤元素。

  • 查找长度严格大于的字符串列表中的那些元素 3.

  • 将 1.0 添加到浮点数列表的每个元素。

  • 给定一个字符串列表 strs 和另一个字符串 sep，生成字符串 包含由 sep 分隔的 strs 的每个元素。例如，给定 输入 ["hi";"bye"] 和 ","，产生 "hi,bye"，确保不产生 在结果字符串的开头或结尾处有一个额外的逗号。

Exercise: association list keys [★★★]

回想一下，关联列表是字典的实现： 一个对的列表，其中我们将每对的第一个组成部分视为键，并且 第二个分量作为值。

编写一个函数 keys: ('a * 'b) list -> 'a list 返回一个列表 关联列表中的唯一键。由于它们必须是唯一的，因此任何值都不应该 在输出列表中出现多次。值输出的顺序不 很重要。你的解决方案能够达到多么紧凑和高效的程度？你能做到吗 一条线和线性空间和时间？ 提示：List.sort_uniq.

Exercise: valid matrix [★★★]

数学矩阵可以用列表表示。在行专业中 表示，这个矩阵

\[\begin{split} \begin{bmatrix} 1 & 1 & 1 \\ 9 & 8 & 7 \end{bmatrix} \end{split}\]

将表示为列表 [[1; 1; 1]; [9; 8; 7]]。让我们代表一个 行向量 作为 int list。例如，[9; 8; 7] 是一个行向量。

一个有效矩阵是一个 int list list ，它至少有一行，至少有一个 列，其中每列具有相同的行数。有很多 int list list 类型的值无效，例如，

  • []

  • [[1; 2]; [3]]

实现一个返回的函数 is_valid_matrix: int list list -> bool 输入矩阵是否有效。对函数进行单元测试。

Exercise: row vector add [★★★]

为以下对象实现函数 add_row_vectors: int list -> int list -> int list 两个行向量的按元素相加。例如，添加 [1; 1; 1] 和 [9; 8; 7] 是 [10; 9; 8]。如果两个向量不具有 相同数量的条目，你的函数的行为是 未指定—也就是说，它可以做任何你喜欢的事情。 提示：有一个 使用 List.map2. 的优雅单行解决方案对函数进行单元测试。

Exercise: matrix add [★★★]

实现一个函数 add_matrices: int list list -> int list list -> int list [matrix addition][matadd] 的列表。如果两个输入矩阵不相同 大小，行为未指定。 *提示：有一个优雅的单行解决方案 使用 List.map2和add_row_vectors`.* 对函数进行单元测试。

Exercise: matrix multiply [★★★★]

实现一个函数 `multiply_matrices: int list list -> int list list -> int matrix multiplication 的列表列表。如果两个输入矩阵是 大小不能相乘，行为未指定。单位 测试函数。 提示：定义矩阵转置和行的函数 向量点积。
