### 2. ❌ 表达和语义问题（HIGH 优先级）

[terminal-tutorial]: https://ubuntu.com/tutorials/command-line-for-beginners

#### 2.1 成语和习语翻译不当

| 问题 | 原文 | 初稿 | 应改为 | 影响 |
|------|------|------|--------|------|
| Idiom 错误 | before we dive in | 潜水之前 | 开始之前/深入学习前 | 😱 严重 |
| 过度逐字翻译 | all of which | 其中许多人 | 其中许多 | 😟 中等 |

#### 2.2 完全错译

| 原文 | 初稿 | 应改为 | 备注 |
|------|------|--------|------|
| Building Lists | 建筑清单 | 构建列表 | 机器翻译搞错了 |
| cons operator | 缺点运算符 | cons 运算符 | "cons" 是编程术语，不是"缺点" |
| Map (章节名) | 地图 | Map/映射 | 章节名应保持一致格式 |

#### 2.3 上下文语义不当

**原文**：
```
There's a lot of similarity between those two functions:
- They both pattern match against a list.
```

**初稿**：
```
这两个函数有很多相似之处：
- 它们都针对列表进行模式匹配。
```

**问题**：
- "进行模式匹配" 表述可以，但"针对...进行"有点生硬
- 建议改为"都对列表进行模式匹配" 或"都用模式匹配处理列表"

### 3. ⚠️ 术语一致性问题（MEDIUM 优先级）

#### 3.1 未统一的翻译

| 概念 | 出现形式 | 是否一致 | 建议 |
|------|--------|--------|------|
| cons operator | "cons 运算符"、"缺点运算符" | ❌ 不一致 | 统一为 "cons 运算符"（保留英文） |
| pattern match | "模式匹配"、"针对...进行模式匹配" | ✓ 一致 | 保持现状 |
| head/tail | "head"、"尾部"、"tail" | ⚠️ 混用 | 建议都保留英文："head" 和 "tail"，或统一翻译为"头部"、"尾部" |
| list | "列表" | ✓ 一致 | 保持现状 |

#### 3.2 chapter 标题翻译不统一

```
src/chapters/hop/map.md          → "Map"
zh-cn/chapters/hop/map.md        → "地图"

应该：如果要翻译，应该是"映射"；如果不翻译，保持"Map"
建议：保持 "Map"（因为这是 OCaml 高阶函数的标准术语）
```

### 4. 📝 格式和排版问题（LOW 优先级）

#### 4.1 列表连接不自然

**原文**：
```
In particular,

- **lists** and **tuples**, might feel similar to Python; and

- **records** and **variants**, might feel similar to `struct` and `enum` types
  from C or Java.
```

**初稿**：
```
特别是，

- **列表**和**元组**，可能感觉与Python类似；和

- **记录**和**变体**，可能感觉类似于 `struct` 和 `enum` 类型
来自 C 或 Java。
```

**问题**：
- "来自 C 或 Java" 位置不对，应该在同一行
- "；和" 的连接不自然

**改正**：
```
特别是，

- **列表**和**元组**可能感觉与 Python 相似；
- **记录**和**变体**可能感觉类似于 C 或 Java 中的 `struct` 和 `enum` 类型。
```

#### 4.2 缩进和换行

部分复杂段落的换行不当，导致上下文割裂。
