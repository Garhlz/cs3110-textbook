# CS3110 教科书中文翻译润色计划

**项目状态**：机器翻译初稿质量评估和系统润色

**交付成果**：高质量的中文 Jupyter Book，适合中文读者本地阅读和构建

---

## 📅 项目时间表

### 第一阶段：质量评估和标准化 (1-2 天)
**目标**：建立润色基准、确定问题类型和优先级

#### 1.1 建立术语表
- [ ] 创建 `glossary.yml`，基于 `translation-guidance.md` 的建议
- [ ] 统计现有翻译中术语的使用情况
- [ ] 标记不一致之处
- [ ] 确定专有名词保留英文还是中文的规则

#### 1.2 抽样审查 (每章取 2-3 个文件)
- [ ] 验证 Jupytext front matter 完整性
- [ ] 检查代码块、链接、公式、HTML 是否破损
- [ ] 确认 Jinja substitution 是否原样保留
- [ ] 统计主要错误类型（表达、术语、格式）
- [ ] 计算需要修改的文件比例

#### 1.3 编制风格指南
- [ ] 中英混用规范（OCaml 关键字、库名等）
- [ ] 统一常见表达（"我们" vs "你"、时态选择等）
- [ ] 代码前后说明的语气风格
- [ ] 列表、表格、引用块的格式规范

**产出物**：
- `glossary.yml` - 术语表
- `STYLE_GUIDE.md` - 中文风格指南
- `SAMPLE_REVIEW.md` - 抽样审查报告（问题汇总）

---

### 第二阶段：按章节系统润色 (3-4 周)
**方法**：LLM 受约束润色 + 人工审查

**处理顺序** (基于 `_toc.yml`)：

| 优先级 | 章节 | 文件数 | 预计工作量 |
|-------|------|--------|----------|
| 1️⃣ | cover.md | 1 | 0.5 天 |
| 2️⃣ | preface/ | 2 | 1 天 |
| 3️⃣ | intro/ | 3 | 1.5 天 |
| 4️⃣ | basics/ | 9 | 3 天 |
| 5️⃣ | data/ | 15 | 5 天 |
| 6️⃣ | hop/ | 9 | 3 天 |
| 7️⃣ | modules/ | 6 | 2 天 |
| 8️⃣ | mut/ | 5 | 1.5 天 |
| 9️⃣ | conc/ | 9 | 3 天 |
| 🔟 | correctness/ | 10 | 3.5 天 |
| 🔟🔟 | ds/ | 8 | 2.5 天 |
| 🔟🔟 | interp/ | 5 | 1.5 天 |
| 🔟🔟 | adv/ | 4 | 1.5 天 |
| 🔟🔟 | appendix/ | 3 | 1 天 |
| 🔟🔟 | styles/ | 2 | 1 天 |

**每章处理流程**：
1. 打开原文 (`src/chapters/*/`) 和初稿 (`zh-cn/chapters/*/`)
2. 逐文件对比，修复三类问题：
   - **表达层**：成语用法、连接词、句式结构、语序
   - **技术层**：术语统一、代码说明、专有名词
   - **结构层**：Markdown 格式、特殊标记、排版
3. 记录修改内容
4. 验证特殊元素不被破坏

**输出**：每个文件一份修改日志（可选）

### 第三阶段：全局验证和质量保证 (3-5 天)
- [ ] 生成术语使用索引，检查一致性

- [ ] 运行 `jupyter-book build zh-cn/`，确保无错误
- [ ] 对比构建后的 HTML，检查特殊元素渲染

**产出物**：
- `GLOSSARY_INDEX.md` - 全书术语使用统计
- `QA_REPORT.md` - 质量保证报告
- 构建成功的 `zh-cn/_build/` (可选上传)

---

### 第四阶段：文档和交付 (1 天)
- [ ] 生成完整的修改日志
- [ ] 编制"翻译说明"文档（如有特殊处理）
- [ ] 整理术语表最终版本
- [ ] 生成项目总结报告

**产出物**：
- `TRANSLATION_LOG.md` - 修改日志
- `GLOSSARY.yml` - 最终术语表
- `README_TRANSLATION.md` - 翻译说明（可选）
- `FINAL_REPORT.md` - 项目总结

## 🎓 质量标准

润色后的翻译应满足：

1. ✅ **忠实原意**：不新增、不删减技术事实
2. ✅ **语言自然**：符合中文教材读习惯，避免生硬逐字翻译
3. ✅ **术语统一**：全书同一概念使用同一术语
4. ✅ **结构完整**：所有代码块、链接、公式、特殊标记保留
5. ✅ **格式规范**：Markdown 和 MyST 特殊元素完好无损
6. ✅ **可构建性**：能成功构建 Jupyter Book
