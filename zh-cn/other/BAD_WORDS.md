# 常见错译词列表

用于快速识别需要修改的文件。可结合 grep 工具使用。

## 🔴 严重错译（必须修改）

这些词在整个文档中应该替换为正确的表达。

### 1. 完全错误的翻译

```
原始错误词     →  应改为        →  grep 搜索模式
================================================================================

缺点运算符     →  cons 运算符      grep -r "缺点运算符"
建筑清单       →  构建列表        grep -r "建筑清单"
建筑列表       →  构建列表        grep -r "建筑列表"
地图           →  Map/映射         grep -r "^# 地图\|^## 地图"  (限制标题)
```

### 2. 机器翻译常见错误

```
模式           →  应改为           →  备注
================================================================================

潜水之前       →  开始/深入之前     grep -r "潜水之前"
其中许多人     →  其中许多          grep -r "其中许多人"
进行模式匹配   →  对...进行模式匹配  (可保留，但需要检查上下文)
```

## 🟡 中等问题（需要审查）

这些不是严格的"错译"，但在特定上下文中需要修改。

### 3. 术语不一致

#### cons/head/tail 系列

```
head           出现形式              检查方式
================================================================================
head           "头部" 或保留 "head"   grep -r "\bhead\b" zh-cn/chapters/data/
               (需要检查每一处)

tail           "尾部" 或保留 "tail"   grep -r "\btail\b" zh-cn/chapters/data/

cons           "缺点" vs "cons"      grep -r "\bcons\b"
               （应该是 cons，不是缺点）
```

#### 章节标题翻译

```
标题           当前翻译              应改为             文件
================================================================================
Building Lists 建筑清单              构建列表           zh-cn/chapters/data/lists.md#L1
Map            地图                  Map (保留)         zh-cn/chapters/hop/map.md#L1
Filter         过滤器                Filter (保留)      zh-cn/chapters/hop/filter.md#L1
```

## 📋 修改检查单

### 检查 1: 查找所有"缺点"

```bash
# 搜索"缺点"
grep -r "缺点" zh-cn/chapters/

# 预期结果：应该只有"缺点运算符"这一类词，应改为"cons 运算符"
# 修改命令（示例）：
sed -i 's/缺点运算符/cons 运算符/g' zh-cn/chapters/data/lists.md
```

### 检查 2: 查找所有"建筑"

```bash
# 搜索"建筑"
grep -r "建筑" zh-cn/chapters/

# 预期结果：应该替换为"构建"
# 修改命令（示例）：
sed -i 's/建筑列表/构建列表/g' zh-cn/chapters/data/lists.md
sed -i 's/建筑/构建/g' zh-cn/chapters/data/lists.md
```

### 检查 3: 查找标题中的中文（可能需要保留英文）

```bash
# 列出所有一级标题
grep "^# " zh-cn/chapters/**/*.md | grep -v "^# [A-Z]"

# 检查是否需要改回英文
```

### 检查 4: 查找"其中许多人"

```bash
# 搜索
grep -r "其中许多人" zh-cn/chapters/

# 通常应改为"其中许多"
sed -i 's/其中许多人/其中许多/g' zh-cn/chapters/
```

## 🎯 分类统计

### 按影响范围分类

```
影响范围       错误词                    预计出现次数
================================================================================
全书           cons 相关                 10-15 处
全书           head/tail                 8-12 处
某几章         标题翻译 (Map, Filter 等) 3-5 处
部分文件       表达不自然               每文件 3-10 处
```

### 按修改难度分类

```
难度    处理方式                         预计工作量
================================================================================
低      grep + sed 自动替换              0.5-1 小时
中      grep 找到后手工审查修改          2-3 小时  
高      逐段对比原文，LLM 润色           需要每文件 10-30 分钟
```

## 🔧 快速修改脚本示例

### bash 脚本：批量修改（谨慎使用！）

```bash
#!/bin/bash

# 修改前先备份
cp -r zh-cn zh-cn.backup

# 修改 1: 缺点 → cons
find zh-cn/chapters -name "*.md" -exec sed -i 's/缺点运算符/cons 运算符/g' {} \;

# 修改 2: 建筑 → 构建
find zh-cn/chapters -name "*.md" -exec sed -i 's/建筑列表/构建列表/g' {} \;
find zh-cn/chapters -name "*.md" -exec sed -i 's/建筑/构建/g' {} \;

# 修改 3: 其中许多人 → 其中许多
find zh-cn/chapters -name "*.md" -exec sed -i 's/其中许多人/其中许多/g' {} \;

# 验证修改（示例）
echo "=== 修改后的 cons ==="
grep -r "cons" zh-cn/chapters/ | head -5

echo "=== 修改后的构建 ==="
grep -r "构建" zh-cn/chapters/ | head -5
```

**⚠️ 警告**：使用 sed 的 `-i` 参数会直接修改文件。使用前请备份！

## 🎓 学习资源

### "cons" 的背景知识
- 来自 Lisp 编程语言
- 名字是 "construct" 的缩写
- 在函数式编程中常见
- 不是"缺点"，而是一个**操作符名字**

### "head" 和 "tail"
- Lisp 中的概念
- head：列表的第一个元素
- tail：列表的剩余部分（除了头元素外的子列表）
