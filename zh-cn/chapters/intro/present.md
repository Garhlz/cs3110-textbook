# OCaml 的现在

{{ video_embed | replace("%%VID%%", "JTEwC3HihFc")}}

OCaml 是一种函数式编程语言。函数式编程语言的核心抽象是数学意义上的函数。函数将输入映射到输出；对于相同的输入，总是产生相同的输出。也就是说，数学函数是*无状态的*：它们不维护函数调用之间持续存在的任何状态信息。函数是*一等公民*：你可以将它们作为其他函数的参数，也可以将函数作为函数的返回值。用函数来表达一切，能提供一个统一且简洁的编程模型，比其他编程语言中的过程和方法更容易推理。

*命令式*编程语言（如 C 和 Java）涉及*可变*的状态，这些状态在整个执行过程中会改变。*命令*通过破坏性地改变状态来指定计算过程。过程（或方法）可能有*副作用*，即在产生返回值的同时更新状态。

**可变性的幻觉**是：它看起来容易推理。机器先执行这个操作，再执行那个操作，如此继续。

**可变性的现实**是，虽然机器擅长复杂的状态操纵，但人类不擅长理解它。原因在于可变性破坏了*引用透明性*：即用表达式的值替换表达式而不影响计算结果的能力。在数学中，如果 $f(x)=y$，你可以在看到 $f(x)$ 的地方替换为 $y$。但在命令式语言中你不能这样做，因为 $f$ 可能有副作用，所以在时间 $t$ 计算 $f(x)$ 可能得到一个值，而在时间 $t'$ 计算可能得到另一个值。

人们容易相信机器维护着一个单一的全局状态，并且每次只执行一个操作。计算机系统费尽心力维持这种错觉。但那只是幻觉。实际上存在许多状态，分散在线程、处理器核心、多个处理器和联网计算机中。机器会同时做许多事情。可变性使得对分布式状态和并发执行的推理变得极其困难。

*不可变性*使程序员摆脱了这些顾虑。它提供了构建正确并发程序的强大方法。OCaml 主要是一门不可变语言，就像大多数函数式语言一样。它确实支持使用可变状态的命令式编程，但在本书的大部分章节中我们不会使用这些特性——一方面是因为我们根本不需要它们，另一方面是为了让你"彻底戒掉"一些你可能没有意识到自己依赖的习惯。这种摆脱可变性的自由是 OCaml 能带给你的最大观念转变之一。

## OCaml 的特性

{{ video_embed | replace("%%VID%%", "T-DIW1dhYzo")}}

OCaml 是一种*静态类型*且*类型安全*的编程语言。静态类型语言在编译时检测类型错误；一旦检测到类型错误，该语言将不允许执行程序。类型安全语言限制了可以对哪些类型的数据执行哪些操作。在实践中，这可以防止很多低级错误（例如把整数当作函数调用），也能防止很多安全问题：超过一半的计算机紧急响应小组（CERT，美国负责网络安全的政府机构）报告的攻击事件是由缓冲区溢出引起的，而这在类型安全语言中是不可能发生的。

有些语言（如 Python 和 Racket）是类型安全的但*动态类型*。也就是说，类型错误仅在运行时被捕获。其他语言（如 C 和 C++）是静态类型的但不是类型安全的：它们检查某些类型错误，但不保证不存在所有类型错误。也就是说，运行时不保证不会出现类型错误。还有些语言（如 Java）使用静态和动态类型的组合来实现类型安全。

OCaml 支持许多高级特性，其中一些你可能已经接触过，一些可能是新的：

- **代数数据类型：** 你可以在 OCaml 中轻松构建复杂的数据结构，无需担心指针和内存管理。*模式匹配*（我们很快就会学到）是一个能让你检查数据结构形状的特性，这使得使用这些数据结构更加方便。

- **类型推断：** 你不需要在所有地方都写下类型信息。编译器会自动推断出大多数类型。这能使代码更容易阅读和维护。

- **参数多态性：** 函数和数据结构可以通过类型参数进行参数化。这对于能够重用代码至关重要。

- **垃圾收集：** 自动内存管理使你免于手动分配和释放内存的负担，这是 C 等语言中常见的错误来源。

- **模块：** OCaml 通过模块让你能够轻松构建大型系统。模块用于隐藏实现细节，只暴露接口。OCaml 远远超越了大多数语言的模块系统，它提供了作用于模块的函数，称为*函子*。

## 工业 OCaml

{{ video_embed | replace("%%VID%%", "eNLm5Xbgmd0")}}

OCaml 和其他函数式语言远不如 Python、C 或 Java 流行。OCaml 真正的优势在于语言工具（即编译器、分析器、验证器、证明器等）。这并不奇怪，因为 OCaml 起源于定理证明领域。

这并不是说函数式语言没有在工业中应用。许多[使用 OCaml 的工业项目][ocaml-industry]和 [Haskell][haskell-industry] 等语言的工业应用都存在。Yaron Minsky（康奈尔大学博士 '02）甚至写了一篇关于[OCaml 在金融行业中的应用][minsky]的论文，解释了 OCaml 的特性如何使其成为快速构建有效复杂软件的好选择。
