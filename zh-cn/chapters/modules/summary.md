# 总结

OCaml 模块系统提供了模块化机制，其功能与你在其他语言中见过的机制类似。
但看到这些机制以不同方式出现，应当能帮助你更好地理解它们。
例如，OCaml 的抽象类型和签名提供了类似于 Java 可见性修饰符和接口的抽象机制。
看到同一个思想在两种语言中以相当不同的方式体现，
也会帮助你在未来遇到其他语言时识别出这个思想。

此外，“类型可以是抽象的”这一思想是编程语言设计中的核心思想。
OCaml 模块系统把这个思想非常直接地呈现出来。
其他语言（例如 Java）则因为把它和许多其他特性耦合在一起，
稍微掩盖了这一点。从某种意义上说，每个 Java 类都隐式定义了一个抽象类型
（实际上，每个可见性修饰符 [`public`, `protected`, `private` 和 `default`]
都对应一个抽象类型，这些类型通过子类型关系相关），
而该类的所有方法都是作用于这种抽象类型的函数。

functor 是 OCaml 中的一项高级语言特性，初看可能显得很神秘。
如果你有这种感觉，请记住：functor 实际上只是接收结构作为输入、
并返回结构作为输出的函数。它们之所以看起来不像普通 OCaml 函数，
是因为结构不是 OCaml 中的一等值：你无法编写一个普通函数来接收结构作为输入，
或返回结构作为输出。但 functor 可以做到这一点。

functor 和 include 支持代码重用。许多面向对象特性实现的代码重用，
也可以通过 functor 和 include 实现。
这并不是说 functor 和 include 完全等同于那些面向对象特性：
有些类型的代码重用用某一组特性实现起来可能比另一组更容易。

理解这一点的一种方式是：类扩展可以看作 functor 和 include 的一种非常受限但有用的组合。
扩展一个类，就像编写一个 functor，把基类作为输入，include 它，然后添加新的函数。
但 functor 提供的功能比类扩展更通用，
因为它们可以计算输入结构的任意函数，而不只是局限于某几种扩展形式。

学习 OCaml 模块系统最重要的收获，也许是理解那些超越任何具体语言的模块化方面：
命名空间、抽象和代码重用。见过这些思想在截然不同的语言中如何组合之后，
你在学习下一门语言时会更容易识别它们。
