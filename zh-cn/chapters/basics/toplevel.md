# OCaml 顶层

{{ video_embed | replace("%%VID%%", "3fzrFY-2ZQ8")}}

*toplevel* 就像 OCaml 的计算器或命令行界面，类似于 Java 的 JShell 或交互式 Python 解释器。顶层非常方便，可以在不启动 OCaml 编译器的情况下快速尝试小段代码。但不要过于依赖它，因为创建、编译和测试大型程序需要更强大的工具。其他一些语言会将顶层称为 *REPL*，代表 read-eval-print-loop：它读取程序员的输入，对其求值，打印结果，然后重复。

在终端窗口中，键入 `utop` 以启动顶层。按 Control-D 退出
顶层。你还可以输入 `#quit;;` 并按回车键。请注意，你必须
在那里输入 `#` ：它是对你已经看到的 `#` 提示的补充。


## 类型和值

你可以在 OCaml 顶层输入表达式。以 a 结束表达式
双分号 `;;` 并按回车键。OCaml 随后会求值
表达式，告诉你结果值以及值的类型。例如：

```ocaml
# 42;;
- : int = 42
```

让我们从右向左仔细阅读 utop 的响应：

* `42` 是值。
* `int` 是值的类型。
* 该值未指定名称，因此符号为 `-`。

utop 交互被“硬编码”为本书的一部分。我们必须输入
所有字符：`#`、`-` 等。但是用于编写的基础设施
本书实际上让我们能够编写由 OCaml 求值的代码
书被翻译成 HTML 或 PDF 的时间。从现在开始，通常就是这样
我们会做的。  它看起来像这样：

```{code-cell} ocaml
42
```

第一个带有 `42` 的代码块是我们要求 OCaml 运行的代码。如果
你想将其输入到 utop 中，可以复制并粘贴它。里面有一个图标
块的右上角可以轻松做到这一点。只需记住添加双
末尾加分号。第二个代码块稍微缩进了一点，是
本书翻译时 OCaml 的输出。

```{tip}
如果你在网络浏览器中查看此内容，请查看右上角的下载
图标。选择 `.md` 选项，你将看到原始的
[MyST Markdown][myst] 本书此页的源代码。你会看到
上面第二个示例的输出实际上并不存在于源代码中。
那挺好的！这意味着输出与任何电流保持一致
我们用来构建本书的 OCaml 编译器的版本。这也意味着任何
编译错误可以作为构建本书的一部分进行检测，而不是
亲爱的读者，它们正潜伏在你身边，等待着你去发现。
```

你可以使用 `let` 定义将值绑定到名称，如下所示：

```{code-cell} ocaml
let x = 42
```

让我们再次剖析该响应，这次从左到右阅读：

* 值绑定到名称，因此使用 `val` 关键字。
* `x` 是值绑定到的名称。
* `int` 是值的类型。
* `42` 是值。

你可以将整个输出发音为“`x` 具有类型 `int` 且等于 `42`”。

## 函数

可以使用如下语法在顶层定义函数：

```{code-cell} ocaml
let increment x = x + 1
```

让我们剖析一下这个回应：

* `increment` 是值绑定到的标识符。
* `int -> int` 是值的类型。这是需要的函数类型
`int` 作为输入并生成 `int` 作为输出。将箭头 `->` 视为
  一种将一种价值转化为另一种价值的视觉隐喻
  value&mdash; 这就是函数的作用。
* 该值是一个函数，顶层选择不打印它（因为它
现在已被编译并在内存中具有不容易的表示形式
  适合美观打印）。相反，顶层打印 `<fun>`，即
  只是一个占位符。

```{note}
`<fun>` 本身不是一个值。它只是表示一个不可打印的函数值。
```

你可以使用如下语法“调用”函数：

```{code-cell} ocaml
increment 0
```

```{code-cell} ocaml
increment(21)
```

```{code-cell} ocaml
increment (increment 5)
```

但在 OCaml 中，通常的说法是我们“应用”函数而不是“调用”它。

请注意 OCaml 在是否写括号方面是如何灵活的，并且
不管你是否写空格。第一次学习的挑战之一
OCaml 可以计算出何时实际需要括号。所以如果你发现
你自己有语法错误的问题，一种策略是尝试添加一些
括号。不过，首选的样式通常是在以下情况下省略括号：
他们不需要。因此，`increment 21` 比 `increment(21)` 更好。

## 在顶层加载代码

除了允许你定义函数之外，顶层还接受
*指令*不是 OCaml 代码，而是告诉顶层本身要做的事情
某事。所有指令均以 `#` 字符开头。也许最常见的
指令是 `#use`，它将文件中的所有代码加载到顶层，
就像你将该文件中的代码输入到顶层一样。

例如，假设你创建一个名为 `mycode.ml` 的文件。在该文件中放入
以下代码：

```ocaml
let inc x = x + 1
```

开始顶层。尝试输入以下表达式，并观察
错误：

```{code-cell} ocaml
:tags: ["raises-exception"]
inc 3
```

发生错误是因为顶层还不知道有关 a 的任何信息
函数名为 `inc`。现在向顶层发出以下指令：

```ocaml
# #use "mycode.ml";;
```

请注意，上面的第一个 `#` 字符向你指示顶级提示。
第二个 `#` 字符是你键入的一个字符，用于告诉顶层你是
发出指令。如果没有这个角色，高层会认为你
正在尝试应用名为 `use` 的函数。

现在再试一次：

```{code-cell} ocaml
:tags: ["remove-cell"]
let inc x = x + 1
```

```{code-cell} ocaml
inc 3
```

## 顶层工作流程

将顶层与存储在文件中的代码一起使用时的最佳工作流程是：

* 编辑文件中的代码。
* 使用 `#use` 加载顶层代码。
* 交互式测试代码。
* 退出顶层。  **警告：** 不要跳过此步骤。

```{tip}
假设你想修复代码中的错误。很容易不退出
toplevel，编辑该文件，然后将 `#use` 指令重新发出到同一文件中
顶层会议。抵制这种诱惑。加载的“过时代码”
同一会话中较早的 `#use` 指令可能会导致令人惊讶的事情
无论如何，当你第一次学习这门语言时，会发生&mdash;令人惊讶的事情。所以
**在重新使用文件之前始终退出顶层。**
```
