{{ ex3 | replace("%%NAME%%", "fib fast")}}

你的 `fib` 实现计算第 50 个斐波那契数的速度有多快？
如果它几乎是即时计算的，那么恭喜！但递归的
大多数人一开始想出的解决方案似乎会无限期地挂起。
问题是显而易见的解决方案会重复计算子问题。对于
例如，计算 `fib 5` 需要计算 `fib 3` 和 `fib 4`，并且如果
这些是单独计算的，大量的工作（实际上是指数量）
正在重做。

创建一个仅需要线性工作量的函数 `fib_fast`。 *提示：*
编写一个递归辅助函数 `h : int -> int -> int -> int`，其中
`h n pp p` 定义如下：

- `h 1 pp p = p`，以及

- `h n pp p = h (n-1) p (pp+p)` 对于任何 `n > 1`。

`h` 的想法是假设前两个斐波那契数是 `pp`
和 `p`，然后向前计算 `n` 更多数字。因此，`fib n = h n 0 1` 为
任何 `n > 0`。

`n` 的第一个值是多少，其中 `fib_fast n` 为负数，表示
发生了整数溢出？
